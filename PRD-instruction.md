# PRD: Claude Code 마스터 가이드 웹사이트

## Context
Claude Code를 처음 접하는 입문자부터 기업 AI 전환을 주도하는 고급 사용자까지,
단계별로 학습할 수 있는 한국어 최초 마스터 가이드 웹사이트를 구축한다.
강의, 컨설팅, 기업 내부 교육, 전자책 출간, 개인 브랜딩 강화의 다목적 플랫폼으로 활용된다.

---

## 1. 제품 비전

> "Claude Code AI 전환 플레이북 — 설치부터 기업 AI 팀 구축까지"

- 한국어 최초 Claude Code 종합 마스터 가이드
- 기술서 + 비즈니스서를 동시에 노리는 포지셔닝
- 웹사이트 → 전자책 → 강의 → 컨설팅으로 이어지는 수익 파이프라인의 최상단

---

## 2. 타겟 독자 페르소나 (3가지 진입 경로)

| 페르소나 | 설명 | 진입 경로 |
|---------|------|---------|
| 입문자 | 비개발자, 처음 AI 도구 접하는 직원 | 레벨 1부터 설치 포함 전체 학습 |
| 개발자 | 주니어~시니어 개발자, 기존 도구 경험 있음 | 설치 스킵, 중급 콘텐츠부터 |
| 기업 의사결정자 | CTO, 팀장, 스타트업 창업자 | 마스터/비즈니스 임팩트 섹션 직접 진입 |

홈페이지에 3가지 진입 경로 명시. 각 챕터에 전제 지식 배지 + 스킵 링크 제공.

---

## 3. 커리큘럼 구조 (5레벨 + 캡스톤)

### 레벨 1 — 입문 (Beginner)
- Claude Code란? 왜 써야 하는가 (vs Cursor, Copilot 비교)
- 설치: Node.js → npm install -g @anthropic-ai/claude-code
- API 키 발급 및 환경변수 설정
- 플랫폼별 주의사항 (Windows/Mac/Linux)
- IDE 통합: VS Code 확장, JetBrains 플러그인
- 터미널에서 첫 번째 claude 실행
- 기본 명령: 파일 읽기, 코드 작성, 버그 수정 실습

### 레벨 2 — 기초 (Foundation)
- CLAUDE.md로 프로젝트 컨텍스트 설정
- 권한 모드 이해 (수동 승인 vs 자동 승인)
- Git 연동: commit 메시지 작성, PR 생성 자동화
- 빌트인 슬래시 커맨드 완전 정복 (/help, /clear, /compact)
- 컨텍스트 관리: 토큰 절약 전략, 긴 세션 관리
- 메모리 시스템: MEMORY.md, 세션 간 기억 유지
- 멀티파일 작업 전략

### 레벨 3 — 중급 (Intermediate)
- Hooks 시스템: PreToolUse, PostToolUse, Stop 이벤트
- MCP 서버 연결: GitHub, Slack, Notion, Jira 연동 실습
- 멀티 모델 전략: Opus/Sonnet/Haiku 언제 무엇을 쓸지
- 비용 최적화: API 비용 계산, Prompt Caching 활용
- 테스트 자동화 워크플로우
- CI/CD 통합: GitHub Actions에서 Claude Code 자동화
- 대규모 코드베이스 탐색 전략
- 트러블슈팅: 자주 발생하는 오류, 무한루프 탈출법
- 효과적인 프롬프트 작성법

### 레벨 4 — 고급 (Advanced)
- 커스텀 Skills 만들기: 나만의 슬래시 커맨드 작성
- Headless/자동화 모드 (--print, --output-format json)
- 보안 주의사항: API 키 보호, .env 파일 접근 차단
- Claude Code SDK 소개 및 기초
- 서브에이전트 개념 이해 및 설계 패턴
- Docker 환경에서 Claude Code 실행
- 팀 협업 패턴: 공유 CLAUDE.md, 팀 컨벤션 강제화
- 코드 리뷰 워크플로우: PR 리뷰 자동화

### 레벨 5 — 마스터 (Master)
- Co-work 멀티에이전트: 병렬 Task 오케스트레이션
- Claude Agent SDK 심화
- 커스텀 서브에이전트 설계 패턴
- Claude API 연동 패턴
- Extended Thinking 활용
- 보안, 권한 관리, 프로덕션 배포 패턴

### 캡스톤 — 기업 팀 에이전트 시스템 (최종 프로젝트)

```
CEO 오케스트레이터 에이전트
├── 마케팅팀 에이전트  → 콘텐츠 생성, 캠페인 분석, SNS 자동화
├── HR팀 에이전트      → 채용 스크리닝, 온보딩 문서, 정책 Q&A
├── 개발팀 에이전트    → 코드 리뷰, 문서화, 버그 트리아지
├── 재무팀 에이전트    → 보고서 생성, 비용 분석, 예측
├── 영업팀 에이전트    → 제안서 작성, 리드 분류, CRM 업데이트
├── 법무팀 에이전트    → 계약서 검토, 컴플라이언스 체크
└── CS팀 에이전트      → 티켓 분류, 응답 초안, 에스컬레이션
```

업종별 3개 템플릿: 스타트업 / 중소기업 / 엔터프라이즈

## 4. 기술 스택

| 역할 | 선택 | 이유 |
|------|------|------|
| 프레임워크 | Docusaurus | 문서+튜토리얼 특화, i18n 내장, Meta 장기 유지보수 |
| 스타일링 | Docusaurus 기본 테마 + 커스텀 CSS | 빠른 시작 |
| 코드 하이라이팅 | Prism (Docusaurus 내장) | 설정 불필요 |
| 검색 | Algolia DocSearch | 무료 오픈소스 프로젝트 적용 가능 |
| 배포 | Vercel | 최고의 DX, PR preview URL, 내장 Analytics |
| 소스 관리 | GitHub (공개 저장소) | 커뮤니티 기여, 브랜딩, 오픈소스 효과 |
| i18n | Docusaurus i18n | 한국어 우선, 향후 영어 확장 준비 |

## 6. 비즈니스 모델 (수익 파이프라인)

```
웹사이트 (전체 무료) → 이메일 게이트 (레벨 4~5) → 전자책 (유료) → 강의/워크샵 (유료) → 컨설팅
```

| 단계 | 모델 | 목적 |
|------|------|------|
| 레벨 1~3 | 완전 무료 | 독자 확보, SEO, 브랜딩 |
| 레벨 4~5 | 이메일 등록 필요 (무료) | 이메일 리스트 구축 |
| 전자책 | 유료 (웹 내용 + 심화 + 템플릿) | 직접 수익 |
| 최종 프로젝트 템플릿 | 유료 패키지 | 기업 고객 수익 |
| 강의/컨설팅 | 유료 | 고수익 채널 |

## 8. UI/UX 핵심 요소

- 홈페이지 3가지 진입 경로 (입문자 / 개발자 / 기업 담당자)
- 왼쪽 사이드바: 챕터 네비게이션
- 오른쪽: 페이지 내 목차 (TOC)
- 상단 진행률 표시 바
- 복사 버튼이 있는 코드 블록
- 챕터별 난이도 배지
- "이미 알고 있다면 → 건너뛰기" 스킵 링크
- 다크/라이트 모드
- 모바일 반응형
